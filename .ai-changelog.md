# Banana Parser — AI Agent Changelog

> **[ WARN ] Описание файла:** Этот лог изменений (changelog) предназначен для быстрого введения новых AI-агентов в контекст последних изменений проекта. При внесении существенных доработок добавляйте сюда запись.

## [v3.1.0] - 2026-04-20
*Текущий фокус: 100% код-ревью и стабилизация.*
- **Static Analysis & Code Quality**: Проведена масштабная сверка (flake8/pylint). Устранены неиспользуемые импорты (например, в `run_scraper.py`, `auth.py`, `browser_core.py`) и пропущенные f-строки (без плейсхолдеров). 
- **AI Context Setup**: Создан и обновлен единый файл архитектуры `.ai-context.md`, исправлены проблемы с кодировкой в нём. Теперь все агенты могут сразу понимать разницу между DOM парсингом и перехватом GraphQL.
- **Project Structure**: Создан `.ai-changelog.md` файл для ведения машиночитаемой истории. 

## [v3.0.0] - Весна 2026
*Масштабное обновление и ребрендинг.*
- **Rebranding**: Проект FilPars полностью переименован в Banana Parser во всех исходных файлах, `.md` документации, установщиках, и графическом интерфейсе "Liquid Glass".
- **MCP Integration**: Логика парсинга обернута через `mcp_server.py`. Legacy slash-команды перенесены в `.claude/skills/` для корректной регистрации в новом Claude Code окружении.
- **Content Filtering**: Разработана система умной фильтрации на предмет "AI топиков" с высокочастотными стоп- и триггер-словами. Настроена балансировка false positives.
- **Robust Pipeline**: Заменен нестабильный сингл-сорс парсер для Reels на мульти-фаллбэковую систему (IQSaved, Igram, FastDL через Puppeteer). Интегрирован `Whisper` для извлечения текстовых субтитров из "немых" видеопостов: `pending` Reels теперь обрабатываются в очереди. 
- **UI & UX**: Интегрирован "Liquid Glass" GUI. Экспорт в HTML работает более надежно — исправлены проблемы с кодировкой (UTF-8 by default). Добавлен кастомный Scrollbar.

## [v2.5.0] - Зима 2026
*Stealth Architecture.*
- **Data Capture**: Полный отказ от BeautifulSoup в пользу `page.on('response', handle_response)`. Структура `PostData` теперь формируется на лету из JSON/GraphQL трафика.
- **Headless Bypass**: Интегрирован `playwright-stealth`. Добавлен скрипт `auth.py` для ручного прохождения логина и сохранения `storage_state.json`.
