## [v3.2.1] - 2026-04-21 — Критический баг-фикс: launcher.html

**Проблема:** Окно Banana Parser зависало на экране "Инициализация систем..." и не переходило к форме настроек.

**Корневая причина:** Скрипт `remove_emojis.py` (запущенный в сессии v3.1) перезаписал `launcher.html`, удалив весь блок `<form id="configForm">...</form>` из `.scroll-area`. JS-код на строке 429 делал `document.getElementById('configForm').onsubmit = ...` — это бросало `TypeError: Cannot set properties of null`, что прерывало весь JS-bootstrap. В результате `pywebviewready` хэндлер не срабатывал и preloader не скрывался.

**Исправления:**
- `ui_templates/launcher.html` — восстановлен полный блок `<form id="configForm">` из git-бэкапа (`launcher_old.html`, коммит `a5bf644`)
- `ui_templates/launcher.html` — добавлена защита в `syncUI()`: `if (!el) continue; if (!lbl) continue;` — теперь JS не падает при отсутствующих элементах DOM
- `ui_templates/launcher_old.html` — временный файл бэкапа (можно удалить после проверки)
- `.ai-context.md` — полностью переработан: добавлены таблицы файлов, схема потока данных, описание структуры launcher.html, таблица опасных зон

**Файлы изменены:**
- `ui_templates/launcher.html` ← ОСНОВНОЕ ИСПРАВЛЕНИЕ
- `.ai-context.md` ← документация
- `.ai-changelog.md` ← этот файл

## [v3.2.0] - 2026-04-21 — Auth Wall + README промпты

**Изменения:**
- `skills.py::safe_goto()` — добавлен Auth Wall детектор: если Instagram редиректит на `/accounts/login/`, бросается `PermissionError` вместо бесконечного ожидания

- Выполнен `git push origin main` с коммитом `cf08eea`
