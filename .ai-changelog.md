## [v3.3.0] - 2026-04-21 — AI Semantic Classifier + UI Explanations

**Новое:**
- **`ai_classifier.py`** (NEW) — Singleton `TopicClassifier` на базе `sentence-transformers`.
  Модель `paraphrase-multilingual-MiniLM-L12-v2` (118 МБ, один раз). Поддерживает RU/EN.
  `warm_up(topic_text)` в фоновом потоке. `classify(text, threshold=0.35)` → `True/False/None`.
  **Полный graceful fallback** — при отсутствии библиотеки работает как bypass, не падает.

- **`interceptor.py` PostFilter** — добавлены поля: `ai_topic_text`, `ai_enabled`, `ai_threshold`.
  В `matches()`: если keyword match провалился → AI fallback (cosine similarity). Если AI не готов → bypass.

- **`skills.py` master_viral_hunter** — читает AI-параметры из `filters`, запускает `warm_up_in_background()`.

- **`run_scraper.py`** — `getAiStatus()` / `getAiStatusText()` методы для JS, `ai_recognition` и `ai_topic_text` передаются в PostFilter.

- **`launcher.html`** — ключевые слова получили пояснение «Что это: запрос к Instagram». Блок ИИ тематики стал тоглом с раскрывающимся телом + live-индикатор «AI загружается... / AI готов». JS: `pollAiStatus()` каждые 2с, `ai_recognition` сохраняется в localStorage.

- **`.claude/skills/banana-keywords/SKILL.md`** (NEW) — скилл `/banana-keywords` для составления точных ключевых слов ниши (3 уровня: бренды → термины → контекст).

- **`requirements.txt`** — добавлен `sentence-transformers`.

**Файлы изменены:** `ai_classifier.py` (new), `interceptor.py`, `skills.py`, `run_scraper.py`, `ui_templates/launcher.html`, `.claude/skills/banana-keywords/SKILL.md`, `requirements.txt`, `CLAUDE.md`, `.ai-context.md`

## [v3.2.1] - 2026-04-21 — Критический баг-фикс: launcher.html

**Проблема:** Окно Banana Parser зависало на экране "Инициализация систем..." и не переходило к форме настроек.

**Корневая причина:** Скрипт `remove_emojis.py` (запущенный в сессии v3.1) перезаписал `launcher.html`, удалив весь блок `<form id="configForm">...</form>` из `.scroll-area`. JS-код на строке 429 делал `document.getElementById('configForm').onsubmit = ...` — это бросало `TypeError: Cannot set properties of null`, что прерывало весь JS-bootstrap. В результате `pywebviewready` хэндлер не срабатывал и preloader не скрывался.

**Исправления:**
- `ui_templates/launcher.html` — восстановлен полный блок `<form id="configForm">` из git-бэкапа (`launcher_old.html`, коммит `a5bf644`)
- `ui_templates/launcher.html` — добавлена защита в `syncUI()`: `if (!el) continue; if (!lbl) continue;` — теперь JS не падает при отсутствующих элементах DOM
- `ui_templates/launcher_old.html` — временный файл бэкапа (можно удалить после проверки)
- `.ai-context.md` — полностью переработан: добавлены таблицы файлов, схема потока данных, описание структуры launcher.html, таблица опасных зон

**Файлы изменены:**
- `ui_templates/launcher.html` ← ОСНОВНОЕ ИСПРАВЛЕНИЕ
- `.ai-context.md` ← документация
- `.ai-changelog.md` ← этот файл
